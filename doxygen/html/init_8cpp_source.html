70 //
71 // Thread management and startup/shutdown:
72 //
73 // The network-processing threads are all part of a thread group
74 // created by AppInit() or the Qt main() function.
75 //
76 // A clean exit happens when StartShutdown() or the SIGTERM
77 // signal handler sets fRequestShutdown, which triggers
78 // the DetectShutdownThread(), which interrupts the main thread group.
79 // DetectShutdownThread() then exits, which causes AppInit() to
80 // continue (it .joins the shutdown thread).
81 // Shutdown() is then
82 // called to clean up database connections, and stop other
83 // threads that should only be stopped after the main network-processing
84 // threads have exited.
85 //
86 // Note that if running -daemon the parent process returns from AppInit2
87 // before adding any threads to the threadGroup, so .join_all() returns
88 // immediately and the parent exits from main().
89 //
90 // Shutdown for Qt is very similar, only it uses a QTimer to detect
91 // fRequestShutdown getting set, and then does the normal Qt
92 // shutdown thing.
93 //
